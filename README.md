# 聴景（ちょうけい） — 音の観察ノート

> 聴くことで立ち上がる景色を記録するノート。
> 音を分析するのではなく、聴いたあと何が残ったかを書きとめ、他者の景色を借りるための道具。

同じ音を聴いても、人によって見える景色は違う。聴景はその景色を記録し、揺れを静かに見えるようにするPWA。
設計思想は [CONCEPT.md](CONCEPT.md)、データ定義は [SCHEMA.md](SCHEMA.md)。
（旧称「音色のことば」。データ構造・保存先は互換）

## できること（MVP）
- **クイック入力**: 言葉＋良し悪し（ポジ/中立/ネガ）だけで記録。「誰が」「何について」「対比」「メモ」は任意。
- **観察対象（Focus）**: 演奏全体/楽器/解釈/録音/空間/瞬間 をチップで分類（既定=演奏全体）。後の集計・ソート用。
- **Timestamp / 場所メモ**: 任意。Focus=瞬間 で自動表示、他Focusでも開ける（例 `2:14` / `ppの入り`）。
- **連続入力**: 「保存して次の言葉」で source/person/origin/focus/timestamp を保持し、言葉だけ次々入力。
- **再利用**: source/person は「最近」チップとサジェストで再利用（既存sourceを選ぶとURL等も自動紐付け）。
- **URLからの自動入力**: 「何を聴いたか」にYouTube URLを貼るとタイトルを、「誰の視点か」にXプロフィールURLを貼ると `@username`（可能なら表示名）を自動入力。手入力は上書きしない・失敗しても保存可。
- **表記ゆれのまとめ**: 「明るめ」を打つと「明るい にまとめる」を候補表示（自動マージはしない／押して確定）。
- **言葉詳細**: 良し悪しの内訳・誰がどう使ったか・≒近い言葉・人ごとの反対語(⇄)・揺れフラグ・**Focus内訳**・**発話一覧**。
- **バックアップ**: 右上 ⤓ でJSON書き出し、⤒ で読み込み（端末内保存のため定期的に推奨）。旧バックアップも読み込み可。

## 動かす（ローカル）
このフォルダで静的サーバを立てるだけ。ビルド不要。

```
python -m http.server 5050
```

ブラウザで http://localhost:5050 を開く。
（Service Worker / ESモジュールの都合で `file://` 直開きではなく http で開くこと）

## スマホに入れる
1. 下記いずれかで公開してURLを得る。
2. スマホのブラウザでURLを開き、「ホーム画面に追加」。アプリのように起動できる。

### 公開のおすすめ（無料・簡単）
- **Netlify Drop**: <https://app.netlify.com/drop> にこのフォルダをドラッグ&ドロップ。
- **GitHub Pages**: リポジトリに push → Settings > Pages で公開。
- **Cloudflare Pages** なども可。いずれも静的ホスティングでそのまま動く。

## 構成
| ファイル | 役割 |
|---|---|
| `index.html` | エントリ |
| `app.js` | 画面（Preact + htm、ビルド不要） |
| `db.js` | データ層（Dexie/IndexedDB）。将来クラウド同期に差し替えやすいよう `repo` に分離 |
| `styles.css` | スタイル（スマホ縦・ダークモード対応） |
| `manifest.webmanifest` / `sw.js` / `icon.svg` | PWA（ホーム追加・オフライン） |

## データの居場所と注意
- 保存先は**そのブラウザのIndexedDB**（端末内のみ）。ブラウザのデータ消去や機種変更で消える。
- 当面は **JSONエクスポートで手動バックアップ**。複数端末同期が必要になったらクラウド同期を追加する設計（`db.js` を差し替え）。
